Pack'n'Roll

Pack'N'Roll is an application to help users plan and pack for upcoming trips.

    Pack'n'Roll was built using:
  • Ruby On Rails
  • JavaScript
  • jQuery
  • HTML/CSS
  • PostgreSQL
    10 API integrated:
  • Exchange Rates API
  • Autocomplete City API
  • Google Maps API
  • TimeZoneDB API
  • Weather Underground API
  • IP Address Geolocation API
  • Facebook Oauth API
  • Google Oauth API
  • Linkedin Oauth API
  • Github Oauth API

See code on Github

or

See app on Heroku

Application Features:

User can create a personal account (sign up / log in) and edit his/her profile settings.

User can create a Trip by adding a destination name, start date and end date of the planned trip and choose the activities planned.

Once the trip is created, user can check current weather for the destination location as well as see the distance between 2 locations on the map.

The trip can be updated or deleted. On the trip page, a user is provided with the list of suggested items to pack (by category). Custom items can be added on the same page. User can mark items as packed and save or print current progress. User has access to a Calendar with all the planned trips to easily find and track past of future travel.

User can check the currency of a searched country and the currency calculator is available. Time difference between his/her current location and the destination is displayed on the world map.

Application Features:

Finding an event. A user has the ability to see all events that match their interests but ONLY during times that they are available. This is done by having the user create time slots when they are "normally free" on each day of the week. This schedule is then duplicated every week. Imromptu gives that specific user events at the beginning of each week. A user can also do custome searches if they suddenly find themselves having some available time during the week, eliminating the need for the user to go in and change their settings.

Hosting an event. Since every user has their "normally free" times included in their profiles, someone who is looking to host an event can see when users (who will remain anonymous), that are interested in their topic, are free. These statistics allow the host to create their event at the most optimal time for the potential guests in the area. The host can also send out an email to all anonymous users as well, informing them of the event and why they should go.
